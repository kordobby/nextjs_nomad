# NextJS Intro
```javascript
NextJS application VS create-react-app application

* React => Everything is " Client-side rendering (CSR) "
  - Users has to see "white screen" => becuz Browser has to download the whole ReactJS code and then ReactJS has 2 render everything in the FE.
* NextJS => in NextJS application, pages r pre-generated!
  - So when the users go to a page generated by NextJS, the init State of ur components is going to b rendered automatically. (going to be rendered on the HTML code)

** REHYDRATION STEP?
  - NextJS will run ReactJS on the BE. NextJS will pre-generate that page, and it will be HTML page.
  - there's no white screen or loading stage => but after the user downloads all the JS, then ReactJS will take-over. and everything will become a ReactJS application.
```

```javascript
* 라이브러리 vs 프레임워크
  * 라이브러리와 프레임워크의 주요 차이점은 "Inversion of Control"(통제의 역전)
  * 라이브러리 : 개발자가 어떤 프로그램을 가져다 쓰는것
    - 사용자가 파일 이름이나 구조 등을 정하고, 모든 결정을 내림
    - 라이브러리에서 메서드를 호출하면 사용자가 제어할 수 있음
    - ex) React : 렌더링할 때 ReactDOM.render() 를 불러와서 사용
  * 프레임워크 : 정해진 틀 안에서 커스터마이징
    - 파일 이름이나 구조 등을 정해진 규칙에 따라 만들고 따름
    - 프레임워크에서는 제어가 역전되어 프레임워크가 사용자를 호출
    - ex) NextJS : 정해진 규칙에 따라 코드를 작성하면 렌더링됨
```
